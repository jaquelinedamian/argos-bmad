## Nome do projeto
Argos Saúde

## Repositório
argos-bmad

## Objetivo do BMAD neste estágio

Apoiar a definição de interfaces, fluxos, estrutura funcional, documentação e organização do recorte acadêmico Home Care 4.0, dentro do contexto mais amplo do ecossistema Argos Saúde.

## Diretriz estratégica

Embora o Argos Saúde seja uma plataforma com visão de startup e potencial de expansão para múltiplos módulos, o foco operacional imediato dos agentes deve ser o TCC.

Os agentes devem priorizar:
- clareza funcional
- jornadas por papel
- arquitetura de informação
- estrutura de telas
- componentes de interface
- documentação de produto
- organização modular

## Modelo do sistema

O sistema adota uma identidade unificada com múltiplos papéis.

Existe um único usuário, que pode possuir um ou mais papéis simultaneamente, como:
- paciente
- prestador de serviço
- administrador
- familiar ou cuidador

As permissões, ações, interfaces e fluxos disponíveis dependem do papel ativo no momento da navegação.

## Domínios principais do sistema

- identidade e acesso
- usuários e papéis
- pacientes e vínculos
- monitoramento de saúde
- alertas
- agenda
- documentos
- comunicação
- marketplace de profissionais
- administração

## Regra operacional para os agentes

Os agentes não devem expandir o projeto desnecessariamente.

Sempre que houver dúvida entre visão futura e entrega imediata, deve prevalecer o foco no TCC.

## Papel esperado dos agentes

Os agentes devem atuar como uma equipe colaborativa composta por:
- especialista em produto
- gerente de projeto
- especialista em UX
- especialista em UI
- especialista em fluxos
- especialista em arquitetura funcional
- especialista em front-end

## Resultado esperado

Gerar base clara, coerente e reutilizável para:
- criação de Specs
- elaboração de fluxos
- construção de wireframes e interfaces
- refinamento no Figma
- futura implementação do sistema
