# Frontend Rules

- sempre usar TypeScript
- nunca gerar JavaScript puro
- sempre pensar mobile first
- sempre considerar acessibilidade
- toda tela deve ter estados de loading, empty, error e blocked
- sempre componentizar
- sempre nomear componentes de forma clara
- não misturar regra de negócio pesada com componente visual
- usar dados mockados quando necessário
- seguir estritamente os Specs do projeto
- não criar telas fora do Screen Map
